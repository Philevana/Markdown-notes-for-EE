# 第3章 3.4 常见数值求解器

## 1. 本章导读
### 1.1 学习目标
本节目标是把“优化模型”与“可运行求解器”连接起来。学完后，你应能识别 QP/SOCP 这类标准问题形式，理解 SQP 与内点法的基本思想，知道不同算法在精度、速度、规模和鲁棒性上的权衡，并能据此为机器人控制、轨迹规划和自动驾驶约束优化任务选择合适求解器。

### 1.2 本章结构
本节先介绍常见标准问题（QP、SOCP）及其建模方式，再讲 SQP 与内点法的核心迭代机制，随后讨论工程实践中的收敛判据、数值稳定与实时性问题。最后通过例题、分层习题和工程练习形成完整应用链条。

## 2. 知识讲解
### 2.1 QP 与 SOCP：从模型到标准形式
**定义**：二次规划（QP）通常写为
$$
\min_x\ \frac12x^\top Hx+f^\top x,
\quad \text{s.t. } Ax\le b,\ Ex=d,
$$
其中 $H\succeq0$ 时问题是凸 QP。QP 在工程中极其常见，例如控制分配、带约束最小二乘、MPC 子问题等。若目标和约束结构可转成二次+线性形式，通常优先考虑 QP 求解器，因为工业工具链成熟、求解速度稳定。

**定义**：二阶锥规划（SOCP）一般写成
$$
\min_x\ c^\top x,
\quad \text{s.t. } \|A_i x+b_i\|_2\le c_i^\top x+d_i,
\ Ex=f.
$$
SOCP 可表达许多“范数约束”问题，如鲁棒回归、摩擦锥近似、某些安全边界约束。与 QP 相比，SOCP 表达能力更强，但计算代价通常更高。

建模时要注意“标准化”思路：先识别目标与约束是否能映射到现有求解器支持的标准形式，再决定算法。许多求解失败并非算法太弱，而是问题缩放不当或约束表达不规范。

### 2.2 SQP：把非线性约束优化分解为一系列 QP
**定义**：序列二次规划（SQP）是一类求解非线性约束优化的方法。其思想是在当前迭代点对约束做一阶线性化，对拉格朗日函数做二次近似，得到一个 QP 子问题；解出步长方向后再更新原变量与乘子。

典型步骤是：
1. 在 $x_k$ 处构造 QP 近似子问题；
2. 解子问题得到步长 $p_k$；
3. 结合线搜索或信赖域更新 $x_{k+1}=x_k+\alpha_k p_k$；
4. 更新 Hessian 近似（常用 BFGS）与乘子估计。

SQP 的优势是局部收敛速度快、约束处理自然，尤其适合中等规模、精度要求较高的非线性问题。局限在于对初值敏感，且每步需要解 QP，若子问题病态或不可行，算法容易出现停滞。工程上通常配合可行性恢复、阻尼策略和约束松弛提高鲁棒性。

### 2.3 内点法：从障碍函数到原始-对偶迭代
**定义**：内点法通过在可行域内部迭代并引入障碍项处理不等式约束。例如把约束 $g_i(x)<0$ 融入目标为
$$
\phi_t(x)=tf(x)-\sum_i \log(-g_i(x)),
$$
其中 $t$ 逐步增大。现代求解器更常采用原始-对偶内点框架，同时更新原变量、对偶变量与松弛变量，沿中心路径逼近最优解。

内点法的优势是对大规模稀疏凸问题表现稳定，精度可控，且适合统一处理 QP/SOCP/LP 等标准锥问题。其挑战在于每次迭代需解线性方程组，对数值线性代数质量要求高；若问题缩放差或条件数恶化，迭代可能变慢。

在机器人与自动驾驶实践中，内点法常用于离线高精度求解或中速在线优化；对于极高频率控制环，常需配合问题简化、热启动和结构利用（稀疏性、块对角）才能满足实时性。

## 3. 例题与习题
### 3.1 例题（讲解型）
考虑凸 QP：
$$
\min_x\ \frac12x^\top x,
\quad \text{s.t. }\mathbf{1}^\top x=1,\ x\ge0.
$$
该问题可解释为“在概率单纯形上最小化二范数平方”。由对称性可知最优点是各分量均分，即
$$
x_i^*=\frac1n\ (i=1,\dots,n).
$$
这个例子展示了：当模型是标准凸 QP 时，求解器不仅能快速给出数值解，也可通过结构直观验证合理性。

### 3.2 基础习题
1. 写出 QP 与 SOCP 的标准形式，并说明主要差异。
2. 给出一个典型可写成 QP 的控制问题（文字描述即可）。
3. 为什么 $H\succeq0$ 对凸 QP 很关键。
4. SQP 每步为什么要解一个 QP 子问题。
5. 内点法中的障碍项起什么作用。
6. 解释“热启动”在序列优化中的意义。
7. 说明数值缩放对求解器收敛的影响。
8. 什么时候优先选 QP 而不是 SOCP。

### 3.3 提高习题
1. 讨论 SQP 与内点法在非线性约束问题中的优缺点与适用边界。
2. 对一个小型非线性约束问题，写出 SQP 的线性化子问题形式。
3. 解释原始-对偶残差在内点法终止判据中的作用。
4. 结合稀疏结构，说明为何线性代数实现常决定求解器性能上限。

### 3.4 工程习题（可结合代码/仿真）
1. 用 Python（如 CVXPy + OSQP/ECOS）分别求解一个 QP 与一个 SOCP，比较求解时间和迭代次数。
2. 在机器人控制分配场景中，构建带输入限幅的 QP，观察不同权重对力矩分配的影响。
3. 在自动驾驶速度规划中添加范数型安全约束，形成 SOCP，并评估实时性。
4. 对同一非线性约束问题，分别用 SQP 与内点法求解，比较初值敏感性与收敛稳定性。

## 4. 面试映射
### 4.1 高频问题
常见问题包括：QP、SOCP 各适合什么类型约束；为什么很多工程问题最终要“标准化建模”；SQP 与内点法核心差别是什么；求解器失败通常是算法问题还是建模/缩放问题；如何在精度与实时性之间做取舍。

### 4.2 回答要点
建议采用“问题结构—算法机制—工程权衡”三段式。先说明模型属于哪类标准问题，再解释对应算法如何迭代，最后给出落地选择依据：频率要求、问题规模、稀疏结构、可接受精度和鲁棒性需求。

### 4.3 项目结合问法
可结合你的项目说明：你如何把原始约束转成 QP/SOCP；为什么选择某个求解器（如 OSQP、qpOASES、ECOS）；怎样处理求解失败（缩放、松弛、热启动、降阶）；最终带来了哪些指标改善（求解时延、可行率、稳定性）。

## 5. 习题答案（简版）
### 5.1 基础题答案
1. QP 是二次目标+线性约束；SOCP 含二阶锥（范数）约束。
2. 例如最小二乘控制分配加执行器上下限。
3. 保证目标凸性与全局最优可解性。
4. 因其本质是局部二次近似 + 线性化约束。
5. 防止迭代跨出不等式可行域边界。
6. 利用上一时刻解缩短当前迭代。
7. 缩放差会恶化条件数并降低收敛稳定性。
8. 当约束可线性表示且实时性优先时通常选 QP。

### 5.2 提高题答案
1. SQP 局部快但初值敏感；内点法稳健但单次线性代数代价较高。
2. 在当前点做约束一阶展开、目标二次近似即可写出。
3. 原始/对偶/互补残差共同衡量是否接近 KKT 解。
4. 大规模问题中线性系统求解器和稀疏利用常是瓶颈。

### 5.3 工程题答案
1. QP 常更快，SOCP 表达更强但开销通常更大。
2. 权重变化会改变误差分配与执行器利用率。
3. 范数约束提升安全性但可能增加求解时延。
4. 实测可比较收敛失败率、迭代次数与时间抖动。
